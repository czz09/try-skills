# 🗺️ try-skills 使用路线图

## 这是给谁看的

你想学一样东西——但不知道从哪开始，不知道方法对不对，不知道每天该做什么，学完还容易忘。

6 个 try 工具，覆盖从目标到巩固的完整路径。

---

## 快速入口

```
┌──────────────────────────────────────────────────┐
│  你现在的状态 → 看这张表找到入口                    │
├──────────────────────────────────────────────────┤
│  有一个大目标，想知道多久能完成                     │ → try-plan
│  太简单了提不起劲 / 太难了想放弃                     │ → try-stretch
│  不知道怎么学                                      │ → try-method
│  有目标但不知道今天做什么                            │ → try-task
│  学完就忘了                                        │ → try-review
│  不确定自己是不是在用对的方法复习                     │ → try-review
└──────────────────────────────────────────────────┘
```

---

## 完整路径（从零开始）

### Step 0：你有一个目标

比如：**「我想在 3 个月内学会用 Python 做数据分析。」**

但不确定这个目标现不现实。

### Step 1：可行吗？→ `try-plan`

```
目标：Python 数据分析
阶段拆分：
  1. 基础语法        → 约 40 小时
  2. Pandas 库       → 约 30 小时
  3. 做一个小项目     → 约 30 小时
  总计：约 100 小时

每天可用：1 小时
预计：100 天 ≈ 3.3 个月

→ 偏紧，需要周末加一点
```

**输出：** 可行/不可行 + 建议。没有详细计划，只看方向对不对。

---

### Step 2：难度对吗？→ `try-stretch`

你可能刚开始学 Python——是觉得「hello world 太简单了没意思」，还是看到函数就头疼？

```
定位结果：恐慌区
判断：函数还没搞懂就想做数据分析，跨度太大

下一步：回退到「看懂一个函数→抄一个函数→改一个函数」
```

**输出：** 你在拉伸区的哪个位置 + 往哪走一步。

---

### Step 3：方法对吗？→ `try-method`

编程是「动手/技能」类——你却在看书而不是写代码。

```
匹配结果：动手/技能 → 模仿→改参→独立

建议：不要看教程，找一个小项目，完整抄一遍，
      改一个变量，再自己写一个。
```

**输出：** 你学的东西属于什么类型 + 对应学法。

---

### Step 4：今天做什么？→ `try-task`

你现在知道该「模仿一个项目」了——但今天具体做什么？

```
今日任务：
花 30 分钟，找一个你已经看完的 Python 教程，
把第一个例子不偷看地重写一遍。

完成标志：能跑通，且每一行都知道它在干什么。
```

**输出：** 唯一的一个任务 + 完成标志。

---

### Step 5：学完了怎么巩固？→ `try-review`

你今天终于写完了那个例子——但明天可能就忘了。

```
类型：动手/技能

复习方案：
- 24小时后：关掉教程，凭记忆重写一遍
- 3天后：再写一次，这次改用不同的变量名
- 1周后：把这段代码和之前学的混在一起练

注意：不要看一遍就觉得自己会了。
```

**输出：** 复习方法 + 间隔建议。

## 一个完整流程的例子

```
你：「我想学好英语」

┌─────────────────────────────────┐
│ try-plan                       │
│ → 目标是「能看懂英文新闻」         │
│ → 拆3个阶段：单词(60h)→语法(40h)→阅读(50h) │
│ → 每天1小时，约5个月             │
│ → 可行，但需要每天坚持            │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│ try-stretch                    │
│ → 你在记单词阶段                 │
│ → 觉得背单词枯燥又记不住          │
│ → 舒适区（用APP刷=假装努力）       │
│ → 建议：每天主动回忆20个，不是刷100个 │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│ try-method                     │
│ → 单词属于「知识+技能」混合        │
│ → 建议先当技能学：抄写→发音→回忆   │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│ try-task                       │
│ → 今天：找一篇100词的短文         │
│ → 划出所有不认识的词             │
│ → 查10个，抄3遍，然后不看写1遍    │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│ try-review                      │
│ → 24h后：不看，凭记忆写那10个词  │
│ → 写不出的标记，重新看一遍再默写   │
│ → 3天后：用这10个词造5个句子     │
└──────────────────────────────────┘
```

---

## 使用原则

**1. 一次只用 1 个工具**
不要同时做可行性检查+难度定位+方法匹配。卡在哪就用哪个。

**2. 从一个工具跳到另一个是正常的**
`try-task` 发现任务太难了 → 跳到 `try-stretch` 调难度。
`try-review` 发现方法不对 → 跳到 `try-method` 换方法。

**3. 不需要每次都用完所有工具**
只有新目标才需要走完整流程。日常学习只需要 `try-task` + `try-review`。

---

## 一句话总结

```
try-plan    → 先算账
try-stretch → 再定位
try-method  → 再配对
try-task    → 再动手
try-review  → 再巩固
```

每次卡住了，就问自己：**「我现在卡在哪个环节？」**
